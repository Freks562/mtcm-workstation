# City of St. Petersburg AI RFI — Compliance/Governance Checklist

Mark each item as `Not started`, `In progress`, `Ready`, or `Blocked`.

## Governance and oversight

- [ ] Define human decision authority boundaries for each proposed AI-assisted workflow.
- [ ] Define escalation/override path when AI output is uncertain or contested.
- [ ] Define accountability owners for policy, operations, and exceptions.
- [ ] Define model/change-control governance process for updates after deployment.

## Public-records and records management

- [ ] Confirm records retention obligations for AI-generated and AI-assisted outputs.
- [ ] Confirm audit trail requirements (input, output, reviewer action, final decision).
- [ ] Confirm public-records request handling approach for AI-related artifacts.
- [ ] Confirm redaction/privacy handling rules for releasable records.

## Security and access control

- [ ] Define data classification boundaries for use/non-use in AI workflows.
- [ ] Define access controls and least-privilege enforcement for operators/reviewers.
- [ ] Confirm logging and monitoring requirements for sensitive operations.
- [ ] Confirm incident response path for AI-enabled workflows.

## Data and quality controls

- [ ] Define approved source systems and data quality gating steps.
- [ ] Define hallucination/error handling standards and reviewer checks.
- [ ] Define quality thresholds and service metrics for pilot success.
- [ ] Define fallback/manual process when AI assistance is unavailable.

## Legal/procurement alignment

- [ ] Confirm response language aligns with RFI terms and disclaimers.
- [ ] Confirm no unsupported claims (certifications, past performance, outcomes).
- [ ] Confirm assumptions/dependencies are explicitly disclosed.
